# 风格评估与整改记录

本轮按当前 `style_profile + function_skills + prompt_builder + result page` 链路，对内置风格逐一做了 Codex 风格评审，并把结论直接落到了提示词、技能编排、自动推荐和结果展示里。

## 评估方法

- 逐一检查每个风格的：
  - 结构模板是否会被通用 `TL;DR / 列表腔` 压平
  - 结果页首屏是否与风格阅读方式一致
  - 自动推荐是否能命中典型任务
  - 可视化与版式是否与风格冲突
- 对具有明显节奏感、对象感、口播感或攻防感的风格，优先增强 `style_fidelity`，而不是继续堆通用摘要规则。

## 本轮新增风格

- `纪实风`
- `书信风`
- `播客口播`
- `辩论风`

## 逐一评估结论

| 风格 | 主要问题 | 本轮调整 |
| --- | --- | --- |
| 幽默风趣 | 容易滑向泛化公众号腔 | 保留 `summary_first`，继续强调事实准确与少口水 |
| 简明大纲 | 基本稳定 | 保持高密度结构输出 |
| 儿童友好 | 容易和通俗风重叠 | 保留“5 句话讲清主题 + 类比”定位 |
| 报纸特稿 | 结构稳定，但展示应更像导语 + 分栏 | 继续走 `newspaper` 版式 |
| 海报解读 | 容易过长 | 保持 `poster + enhanced`，强调一句话结论和重点卡片 |
| 书籍章节 | 容易被碎片化 | 保持章节摘要与节中总结，不鼓励过密 bullet |
| 古文书卷 | 容易被通用摘要压平 | 强化书卷守则与题解/纲目/按语阅读方式 |
| 故事风 | 容易一上来就 TL;DR | 抑制 `summary_first`，结果页改为“先看开场” |
| PPT 汇报 | 容易长段化 | 强化“一屏一重点 / 行动项”守则 |
| 论文风 | 容易只剩格式感、缺少边界意识 | 强化“依据与边界”说明 |
| 商业简报 | 容易和 PPT 汇报重叠 | 继续突出“决策服务”而非单纯展示 |
| 教程手册 | 基本稳定 | 结果页改成“先看步骤框架” |
| 演讲稿 | 容易被摘要式开头打断朗读感 | 抑制 `summary_first`，保留开场定调与号召感 |
| 访谈问答 | 容易被改写成普通分析文 | 抑制 `summary_first`，强调问答结构 |
| 评论社论 | 容易立场不够聚焦 | 保持观点先行与证据支撑 |
| 纪实风 | 之前缺失 | 新增事实脉络与现场细节守则 |
| 书信风 | 之前缺失 | 新增对象感与收束语气守则 |
| 播客口播 | 之前缺失 | 新增口播节奏、停顿感与顺口表达守则 |
| 辩论风 | 之前缺失 | 新增主张、论点攻防、反方回应结构 |
| 诗歌风 | 容易被压成“摘要 + 正文” | 继续抑制 `summary_first`，保留分行与留白 |
| 科普风 | 基本稳定 | 继续强调原理、误区与现实意义 |
| 文雅风 | 容易空泛化 | 保持“雅致但可读”的约束 |
| 通俗风 | 基本稳定 | 保持短句、口语化和结论前置 |
| 幼稚童趣 | 容易和儿童友好重叠 | 保留“天真可爱”而非“教学解释” |
| 欢乐风 | 容易和幽默风重叠 | 保留“分享欲与节奏感”区别 |
| 猜谜风 | 容易过早揭晓答案 | 抑制 `summary_first`，结果页改为“先看谜面” |
| 快餐风 | 容易空心化 | 保留“极前置结论 + 重点速看 + 行动建议” |

## 代码落点

  - `frontend/src/lib/functionSkills.ts`

  - `frontend/src/lib/presets.ts`
- 提示词编排：
  - `backend/app/services/prompt_builder.py`
  - `backend/app/services/style_prompt_optimizer.py`
- 风格感知结果展示：
  - `frontend/src/lib/stylePresentation.ts`
  - `frontend/src/components/ResultViewer.vue`
  - `frontend/src/components/DiscoverViewer.vue`

## 后续建议

- 如果后续要继续做评测，优先按四类风格补样本：
  - 叙事/节奏型：故事风、诗歌风、书信风、播客口播
  - 结构/决策型：PPT 汇报、商业简报、辩论风、快餐风
  - 研究/证据型：论文风、评论社论、纪实风、报纸特稿
  - 执行/教学型：教程手册、科普风、儿童友好、通俗风
